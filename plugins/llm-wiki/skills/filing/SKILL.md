# Filing 워크플로우

LLM 과의 대화에서 생성된 가치 있는 분석 결과를 ONDA product-docs 위키에 보존한다.

## Phase 0: product-docs 경로 해석 및 cwd 전환

이 스킬은 `product-docs` 리포 루트를 기준으로 동작한다. 호출 시 cwd 가 product-docs 가 아닐 수 있으므로 다음 순서로 해석한 뒤 `cd` 한다:

1. `$PRODUCT_DOCS_PATH` 환경변수
2. `$TPORT_HOME` 환경변수 → `$TPORT_HOME/product-docs`
3. 후보 경로 순차 확인:
   - `~/Workspaces/tport/product-docs`
   - `~/dev/tport/product-docs`
   - `~/code/tport/product-docs`
   - `~/projects/tport/product-docs`
4. 모두 실패 시 사용자에게 경로 문의 후 중단

해석된 경로에 `cd` 한 뒤 `git pull origin main --ff-only` 로 최신화.

## 트리거 조건

- "이거 위키에 Filing 해줘", "이 분석 결과 저장해줘"
- "이 비교표 문서로 만들어줘", "이 정리 내용을 프로젝트에 반영해줘"
- "방금 만든 거 보존해줘"

## Filing 대상

다음과 같은 분석 결과가 Filing 대상:

- **비교 분석표**: 경쟁사 분석, 기술 비교, 솔루션 비교 등
- **데이터 기반 인사이트**: 수치 분석, 트렌드 정리 등
- **의사결정 근거**: 기술 선정, 아키텍처 결정에 활용된 분석
- **시스템 개요**: 여러 문서를 종합한 정리
- **전략 분석**: 시장 분석, 포지셔닝 정리 등

**Filing 하지 않는 것:**
- 일회성 질문에 대한 단순 답변
- 단순 조회 결과 (파일 목록, 상태 확인 등)
- 작업 지시에 대한 응답
- 이미 위키에 존재하는 내용의 중복

## 사전 확인

Filing 을 시작하기 전에 사용자에게 확인:

1. **Filing 할 내용**: 대화에서 어떤 부분을 보존할 것인지 (사용자가 명시하지 않으면 제안)
2. **대상 위치**: 어느 프로젝트 또는 제품에 저장할 것인지
3. **문서 유형**: 어떤 유형으로 저장할 것인지

### 문서 유형 매핑 가이드

| 분석 결과 유형 | 권장 문서 유형 | 접두어 | 저장 위치 |
|---|---|---|---|
| 경쟁사/기술 비교 | 참고 자료 | `REF` | `reference/` |
| 의사결정 근거 | 결정 기록 | `DEC` | `decisions/` |
| 아키텍처 정리 | 아키텍처 문서 | `ARCH` | `architecture/` |
| 기능 분석 | 기능 문서 | `FEAT` | `features/` |
| API 정리 | API 문서 | `API` | `api/` |
| 데이터 모델 정리 | 데이터 모델 | `DATA` | `data/` |

## 실행 절차

### 1단계: 내용 정제

대화에서 생성된 분석 결과를 문서 형태로 정제:

- 대화체를 문서체로 변환
- 구조화 (헤더, 테이블, 목록 등 활용)
- 불필요한 맥락 제거, 핵심 내용만 보존
- 출처/근거가 있으면 명시

### 2단계: 문서 생성

해당 프로젝트/제품 폴더에 문서 생성:

- `CLAUDE.md` 의 파일명 규칙 준수
- 해당 문서 유형의 템플릿 참고
- 문서 상태는 `DRAFT` 로 시작
- 원본 대화의 맥락을 간략히 기록 (어떤 질문에서 시작되었는지)

### 3단계: 교차 참조 추가

- 관련 기존 문서에 새 문서로의 링크 추가
- 새 문서에서 관련 기존 문서로의 링크 추가
- 프로젝트/제품 README 의 문서 목록에 추가

### 4단계: 기록

- 해당 프로젝트/제품의 `changelog/` 파일에 변경 기록
- `LOG.md` 상단에 `query` 유형 엔트리 추가:

```markdown
## [YYYY-MM-DD] query | [분석 제목]

- `[생성된 문서 경로]` 생성 — [간략한 내용 설명]
- `[업데이트된 문서 경로]` 교차 참조 추가
```

## 완료 보고

- 생성된 문서의 경로와 간략한 내용 요약
- 추가된 교차 참조 목록
- 후속 보강이 필요한 부분이 있으면 안내

## 주의사항

- 기존 문서와 중복되는 내용이 있으면 새 문서 생성 대신 기존 문서 보강을 제안
- Filing 전 반드시 기존 문서 검색으로 중복 여부 확인
- 대화 맥락 없이도 독립적으로 읽을 수 있는 문서로 정제
- 작업 완료 후 사용자에게 git commit/PR 진행 여부 확인 (이 스킬은 자동 commit 하지 않음)
